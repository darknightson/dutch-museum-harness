# 커밋 스킬

## 트리거
사용자가 "커밋", "커밋해줘", "commit" 이라고 말하면 이 스킬을 실행한다.

## 수행 절차

### 1단계: 변경사항 분석
```bash
git diff --staged
```
스테이징된 변경사항이 없으면:
```bash
git diff
```
로 변경사항을 보여주고 "스테이징할 파일을 선택해주세요"라고 안내한다.

### 2단계: 커밋 메시지 생성

아래 형식을 **정확히** 따른다:
```
<type>(<scope>): <제목 50자 이내>

What:
  - 구체적 변경사항 1
  - 구체적 변경사항 2
Why:
  - 이 변경이 필요한 이유

AI-Use: yes
AI-Model: <사용된 모델명>
AI-Contribution: <기여도>%
```

### type 종류
| type | 용도 |
|------|------|
| feat | 새 기능 |
| fix | 버그 수정 |
| refactor | 리팩토링 |
| docs | 문서 변경 |
| style | 포맷팅, CSS |
| test | 테스트 |
| chore | 빌드, 설정 |

### scope 규칙
- 변경된 모듈/파일의 핵심 영역 (예: analyze, gallery, auth, api)
- 여러 영역이면 가장 주된 것 하나만

### AI-Contribution 기준
| 비율 | 기준 |
|------|------|
| 90~100% | AI가 설계~구현을 자율 수행, 사용자는 지시/승인만 |
| 70~80% | 사용자가 방향/설계를 잡고 AI가 대부분 구현 |
| 40~60% | 사용자와 AI가 공동 작업 |
| 10~30% | 사용자가 직접 작성, AI는 보조 역할 |

대화 맥락을 기반으로 AI가 기여도를 추정한다.
AI를 사용하지 않은 커밋이면 AI-Use: no로 표시하고
AI-Model과 AI-Contribution은 생략한다.

### 3단계: 사용자 확인
생성된 커밋 메시지를 보여주고 확인을 요청한다:
- "이대로 커밋할까요?"
- 수정 요청이 있으면 반영 후 다시 확인

### 4단계: 커밋 실행
확인 받으면:
```bash
git add -A  # (스테이징 안 된 경우에만)
git commit -m "<생성된 메시지>"
```

## 커밋 메시지 예시
```
feat(analyze): 이미지 모드 에픽 생성 품질 개선

What:
  - 이미지 슬라이드에 번호 라벨 추가 [슬라이드 N/총N]
  - 이미지 모드 분석 시 모든 슬라이드 빠짐없이 분석하도록 지시 강화
  - 다중 에픽 모드에 문서 분량 대비 충분한 에픽 생성 지시 추가
  - hallucination 방지: 문서 로드 실패 시 에러 반환
Why:
  - 대용량 Google Slides(122장 등) 이미지 모드에서 에픽이 
    4개만 생성되는 등 분석 품질이 부실한 문제 개선

AI-Use: yes
AI-Model: Claude Opus 4.6
AI-Contribution: 90%
```

## 금지 사항
- 커밋 메시지를 영어로 작성하지 않는다 (제목의 type은 영어, 본문은 한국어)
- 사용자 확인 없이 커밋하지 않는다
- AI-Use 필드를 생략하지 않는다
- 사용자가 수동으로 AI-Contribution를 수정할 수 있다.
